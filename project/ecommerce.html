โปรเจ็คนี้เป็นเกี่ยวกับ E-commerce

โปรเจ็คนี้จัดทำให้เป็นเว็บไซต์ที่มีระบบ E-commerce ที่พัฒนาอยู่สามส่วนด้วยกันคือ frontend backend และ database อย่างแรก frontend เราพัฒนาด้วย framework Angular ส่วน backend พัฒนาด้วย framework NestJs ทั้งสอง framework ใช้ภาษา TypeScript (โปรเจ็คนี้พัฒนามานานแล้ว การดีไซน์จะไม่มีอะไรมากส่วนมากจะเน้นที่ระบบ)

    ระบบที่มีหลักๆเลยคือ

  • ระบบล็อคอิน
  • ระบบสมัครสมาชิก
  • ระบบ token เช็คว่าผู้ใช้ได้ล็อคอินหรือไม่ และป้องกันไม่ให้ผู้ใช้ไปหน้าต่างๆที่เข้าได้เฉพาะแอดมิน
  • ระบบจัดการผู้ใช้สำหรับแอดมิน( อนุมัติให้ผู้ใช้ใช้งาน, ลบผู้ใช้, เปลี่ยนตำแหน่งผู้ใช้ อย่างเช่น เปลี่ยนผู้ใช้เป็นแอดมิน)
  • ระบบจัดการสินค้าสำหรับแอดมิน( ลบสินค้า,แก้ไขรายละเอียดสินค้า)
  • ระบบเพิ่มสินค้าสำหรับแอดมิน( ชื่อ, รายละเอียดสินค้า, ประเภทสินค้า, ราคา, จำนวน, รูปภาพ)
  • ระบบจัดการประเภทสินค้าสำหรับ( เพิ่ม,ลบ,แก้ไข)
  • ระบบจัดการออเดอร์( เมื่อลูกค้าสั่งของจะรอคำอนุมัติจากแอดมิน)
  • หน้าเว็บไซต์สำหรับผู้ใช้( มีสินค้าแนะนำ, สินค้ามาใหม่, หมวดหมู่สินค้า)
  • หน้าเว็บเลือกสินค้าหรือหน้าเว็บดูรายละเอียดสินค้าสำหรับผู้ใช้
  • ระบบเพิ่มสินค้าเข้าตะกร้าสินค้า และหน้าเว็บตะกร้าดูสินค้าที่เลือก
  • ระบบเช็คอิน และหน้าเว็บดูออเดอร์ที่สั่งเพื่อรอการอนุมัติ

Welcome to my project!

This project is an E-commerce website

developed with three main components: frontend, backend, and database. The frontend is built using the Angular framework, while the backend is developed with the NestJs framework. Both frameworks use the TypeScript programming language. Although the project has been developed for a while, the design primarily focuses on system functionality.

    Key features of the system include:

  • Login System: Allows users to log in to their accounts.
  • User Registration System: Enables users to create new accounts.
  • Token System: Checks whether users are logged in and prevents unauthorized access to specific pages, accessible only to administrators.
  • User Management for Admins: Admins can approve user registrations, delete users, and change user roles, such as promoting a user to admin.
  • Product Management for Admins: Admins can delete and edit product details.
  • Add Product for Admins: Admins can add new products with details such as name, description, category, price, quantity, and images.
  • Category Management for Admins: Admins can add, delete, and edit product categories.
  • Order Management: When customers place orders, they wait for approval from admins.
  • User-Facing Website Pages: Includes sections for recommended products, new arrivals, and product categories.
  • Product Selection and Details Page for Users: Users can choose products and view detailed information about each product.
  • Shopping Cart System: Users can add products to their shopping carts, and there's a page to view selected items.
  • Check-in System: Users can check in, and there's a page to view orders awaiting approval.
